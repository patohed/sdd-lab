¿Qué es este repo?

Un laboratorio reutilizable. El producto de ejemplo (un login) es secundario: lo que importa son los artefactos que el equipo va acumulando.

📋

Spec

requirements, architecture, rules, glossary. Lo que hace el sistema.

🧠

Skills

Patrones reutilizables que el agente carga bajo demanda.

🗂️

Decisions

ADRs con el contexto de por qué se eligió cada cosa.

💬

Prompts

Prompts canónicos para cada fase del ciclo SDD.

🎯

Issues (slices)

Vertical slices independientes, cada uno con definition of done.

🤖

AGENTS.md

Instrucciones canónicas que cualquier agente IA lee al iniciar.

El ciclo SDD

Humano arriba, agente al medio, humano al final. La retroalimentación siempre vuelve al spec.

Usar como template

Tres formas de arrancar un proyecto nuevo. Elegí la que mejor te quede.

Recomendado

  1. En GitHub → Use this template.
  2. Nombre + visibilidad de tu repo nuevo.
  3. git clone y abrir en tu IDE.
  4. Editar spec/ para tu dominio. Borrar issues y ADRs de ejemplo.

Cómo "saben" los agentes

No saben mágicamente. Cada IDE lee un archivo distinto. Nuestro repo tiene un solo AGENTS.md canónico, y el resto son stubs de 2 líneas que apuntan ahí.
